Success Response
What to expect when your request is successfully submitted
200 OK Response
json
{
  "QueueID": "12345",
  "Message": "Request submitted successfully"
}

QueueID
A unique identifier for tracking this prescription request. Store this value to check the prescription's processing status later.

Important: Persist the QueueID along with your internal transaction ID. You'll need it to call the status endpoint.
Required Fields Summary
Minimum fields needed for a valid submission
Root Level:

StoreID - Your pharmacy store identifier
VendorName - Your company/integration name
Patient Section:

FirstName - Patient's first name
LastName - Patient's last name
Doctor Section:

DoctorFirstName - Prescriber's first name
DoctorLastName - Prescriber's last name
RxClaim Section:

RxNumber - Prescription number
DrugName - Medication name
Qty - Quantity (as string)
Important Notes
•
Base64 Files: Both PDFFile and DocSignature must be Base64-encoded and ≤ 64 KB each
•
Date Format: Use ISO-8601 format (YYYY-MM-DD) for all dates like DOB and DateWritten
•
All Values as Strings: Send scalar values as strings unless specifically noted otherwise
•
Insurance Optional: The Insurance section can be omitted for cash-pay prescriptions
