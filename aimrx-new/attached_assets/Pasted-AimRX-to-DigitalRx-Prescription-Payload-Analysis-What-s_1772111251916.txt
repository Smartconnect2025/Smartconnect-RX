WHAT'S ACTUALLY ARRIVING AS EMPTY/MISSING (confirmed from Greenwich Rx pass-through monitor)

Even though the code references these fields, they arrive as empty because of data issues and one code bug:

Patient Section:

Field	Expected	What Actually Arrives	Why
PatientStreet	"123 Main St"	missing entirely	Patient's physical_address is null — address was never collected during intake
PatientCity	"Little Rock"	missing entirely	Same reason
PatientState	"AR"	missing entirely	This is the most critical field — Greenwich Rx uses patient state to route to the correct pharmacy location (Rx vs Rx II) based on licensed states. Without it, the system says "DigitalRx not configured"
PatientZip	"72211"	missing entirely	Same reason
PatientPhone	"5015551234"	Shows up	This one works — patients.phone field is populated
Doctor Section:

Field	Expected	What Actually Arrives	Why
DoctorStreet	"400 Clinic Rd"	missing entirely	Provider's physical_address is null — provider never completed their profile address
DoctorCity	"Little Rock"	missing entirely	Same reason
DoctorState	"AR"	missing entirely	Same reason
DoctorZip	"72212"	missing entirely	Same reason
DoctorPhone	"5015556789"	always missing	CODE BUG: line 217 reads provider.phone but the database field is called provider.phone_number — so this will never be sent even when the provider has a phone number on file
DoctorDea	"BS1234567"	never sent	Not in the code at all — the field is not included in the payload. Required for controlled substance prescriptions
RxClaim Section:

Field	Expected	What Actually Arrives	Why
DrugNDC	"00093-0012-01"	"kk225245825"	NDC values in the pharmacy_medications table appear to have invalid/test data
DaysSupply	"30"	never sent	Not in the code at all — the field is not included in the payload

SUMMARY OF ROOT CAUSES

Patient and provider address data is not being collected or is empty — The code tries to read it, but the database fields are null because the information was never entered during signup/intake.

Code bug on DoctorPhone — provider.phone should be provider.phone_number. One-word fix but means doctor phone never transmits.

DoctorDea and DaysSupply are not in the code — These fields were never added to the payload builder.

NDC data quality — The medication catalog has invalid NDC values that don't match real drug identifiers.

Some Store IDs don't match real DigitalRx assignments.

WHAT NEEDS TO HAPPEN

Ensure patient intake collects and requires a physical address (street, city, state, zip)
Ensure provider onboarding collects and requires a physical address
Fix provider.phone → provider.phone_number (line 217 in submit-to-pharmacy)
Add DoctorDea from provider profile to the Doctor payload
Add DaysSupply to the RxClaim payload
Validate and correct NDC values in the pharmacy_medications table
Correct Store IDs in pharmacy_backends table
