SENDGRID EMAIL SYSTEM — COMPLETE INSTRUCTIONS

This app uses SendGrid (@sendgrid/mail npm package) for ALL outgoing emails. Here is exactly how emails work and when they are sent. Replicate this behavior exactly.

Environment variables needed:

SENDGRID_API_KEY — SendGrid API key for sending emails
SENDGRID_FROM_EMAIL — "from" address (default: noreply@aimrx.com)
SENDGRID_FROM_NAME — "from" display name (default: AIM Medical)
INTERNAL_API_KEY — secret key for server-to-server email calls (prevents abuse)
Setup pattern in every file that sends email:

import sgMail from "@sendgrid/mail";
const SENDGRID_API_KEY = process.env.SENDGRID_API_KEY || "";
const FROM_EMAIL = process.env.SENDGRID_FROM_EMAIL || "noreply@aimrx.com";
const FROM_NAME = process.env.SENDGRID_FROM_NAME || "AIM Medical";
if (SENDGRID_API_KEY) {
  sgMail.setApiKey(SENDGRID_API_KEY);
}
If SENDGRID_API_KEY is not set, run in demo mode (log but don't crash):

if (!SENDGRID_API_KEY) {
  return NextResponse.json({
    success: true,
    message: 'Email logged (demo mode - no actual email sent)',
    demoMode: true
  });
}
There are 5 types of emails sent by this app:

1. Payment Link Email (/api/payments/send-payment-email)

When: After a provider creates a prescription and billing generates a payment link
To: Patient email
Subject: "Payment Required: [Medication] Prescription"
Contains: Patient name, medication name, provider name, total amount, payment button/link
Protected by: INTERNAL_API_KEY header (x-internal-api-key) — server-to-server only
Design: Blue gradient header, payment details card with blue-left border, green "Pay Now" button, security notice, 7-day expiry warning
2. Payment Confirmation Email (/api/payments/send-confirmation-email)

When: After patient successfully pays (called from verify-and-complete or webhook)
To: Patient email
Subject: "Payment Confirmed: [Medication] Prescription"
Contains: Transaction ID, medication, provider, pharmacy name, amount paid, fulfillment method (pickup/delivery/shipping), order progress tracker (4 steps), next steps explanation
Protected by: INTERNAL_API_KEY header — server-to-server only
Design: Green gradient header with checkmark, green-left-border details card, 4-step progress tracker (Payment Received checked, rest pending)
3. MFA Verification Code Email (core/services/mfa/mfaService.ts)

When: During login when MFA is enabled for a user
To: User email
Subject: MFA verification code
Contains: 6-digit code, 10-minute expiry
Important: If SENDGRID_API_KEY is not set, MFA returns an error (does NOT run in demo mode — MFA requires real email)
Settings: Code expires in 10 minutes, max 5 attempts, 15-minute lockout after max attempts
4. Welcome/Invite Doctor Email (/api/admin/invite-doctor)

When: Admin creates a new provider account
To: New provider email
Subject: "Welcome to AIM RX Portal - Your Provider Account"
Contains: Login URL (https://app.aimrx.com/auth/login), email, temporary password, next steps (complete profile, add payment, change password), security warning about temporary password
Design: Blue gradient header with logo, login credentials box with blue border, blue "Next Steps" section, yellow security warning
5. Access Request Confirmation Email (/api/access-requests)

When: Someone submits an access request to join the platform
To: Applicant email
Subject: "Thank you for your interest in AIM Medical Marketplace"
Contains: Confirmation that request was received, 24-48 hour review timeline, next steps
Design: Blue gradient header with logo, blue info box explaining next steps
