3. First Principles: Why the Data is Restricted
The PioneerRx API operates on a Stateless Security Model.

The "Secret" is Never Sent: The Shared Secret is never actually included in your API call. It is used locally on your server to create the prx-signature.
Server-Side Verification: When the PioneerRx server receives your request, it looks up your prx-api-key, finds the Shared Secret it has on file for you, and tries to recreate the signature. If they don't match, the request is rejected.
The Risk: Because the Shared Secret is the "Master Key" to your signature, PioneerRx will only send it to you via a secure channel (encrypted email or a secure developer portal), never in a static PDF manual.
